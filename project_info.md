# Архитектура и Дерево Проекта AI_HUB (Next.js)

## 1. Описание проекта
**Название:** AI Bottleneck — Многоканальный ИИ-Анализатор.
Это веб-приложение на базе Next.js, предназначенное для получения максимально точных ответов на запросы пользователя, защищенных от ИИ-галлюцинаций. Приложение отправляет запрос пользователя сразу в 3 различные нейросети (работают параллельно), после чего передает их ответы 4-й нейросети ("Аналитику"), которая анализирует их на наличие ошибок и синтезирует финальный структурированный ответ.

## 2. Технологический стек
* **Фронтенд:** Next.js 14 (App Router), React 18, Tailwind CSS, Radix UI.
* **Стили:** Glassmorphism, темная/светлая тема (next-themes).
* **Бэкенд:** Next.js API Routes (`app/api/chat/route.ts`).
* **Провайдеры ИИ (API):** Groq, OpenRouter (OpenAI SDK).

## 3. Доступные нейросети (По окнам)
В системе (в `lib/i18n.ts`) на данный момент зарегистрированы следующие пулы моделей:

* **Окно 1 (3 модели):** 
  * Llama 3.1 8B (Speed) - groq
  * Gemma 2 9B - groq
  * Gemini 2.0 Flash Lite - openrouter

* **Окно 2 (2 модели):**
  * Mixtral 8x7B - groq
  * Gemini 2.0 Flash - openrouter

* **Окно 3 (2 модели):**
  * Llama 3.3 70B (Versatile) - groq
  * Gemini 2.0 Pro - openrouter

* **Аналитик (2 модели):**
  * DeepSeek R1 (Reasoning) - groq
  * Gemini 2.0 Thinking (Analytic) - openrouter

## 4. Особенности маршрутизации
В `app/api/chat/route.ts` уже есть базовая система **Fallback**: если после 3 попыток выбранная модель отдает ошибку (например, `mixtral` лежит), система автоматически делает запасной запрос к `llama-3.1-8b-instant` и добавляет предупреждение `fallbackWarning`. 
Цель доработки: сделать выбор запасных (Fallback) моделей настраиваемым прямо из пользовательского интерфейса для каждого окна.
