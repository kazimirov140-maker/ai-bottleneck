### 🔍 Что было исправлено:

**1. Проблема с Vercel AI SDK 4.x / 7.x**
При установке зависимостей случайно обновились библиотеки Vercel AI SDK до самых последних версий (7.x / 4.x). Оказалось, что в них полностью переписали метод `useChat` — из него пропали свойства `input`, `setInput`, `isLoading`, `append` и `handleSubmit` (их вынесли отдельно или удалили). Так как весь UI файл `page.tsx` опирается именно на классический API 3.x версии, TypeScript выдавал десятки ошибок об отсутствующих свойствах. 

**Решение:** Вместо того чтобы полностью переписывать всю логику взаимодействия с чатом под новые стандарты AI SDK, я **сделал даунгрейд библиотек** до их совместимых стабильных версий эпохи SDK 3.x:
- `ai@3.4.33` (вместо `ai@7.0.16`)
- `@ai-sdk/google@0.0.55`
- `@ai-sdk/openai@0.0.72`
- `@openrouter/ai-sdk-provider@0.1.0`

Это позволило вернуть `import { useChat } from "ai/react";` и использовать все те же любимые методы для чатов (такие как `m.content`, `chat.isLoading`, `chat.append`).

**2. Асинхронность `streamText` в `route.ts`**
В версии 3.x `streamText` работает немного иначе (в отличие от Vercel AI SDK 4.x) — функция возвращает Promise, поэтому я добавил `await` в вызов стриминга в файле `api/chat/route.ts` и корректный метод ответа `toTextStreamResponse()`:
```typescript
const result = await streamText({
  model,
  messages,
  system,
});
return result.toTextStreamResponse();
```
